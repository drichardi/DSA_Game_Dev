CS 300: Data Structures & Algorithms

Applied Game Development with Unity & C#

Course Overview

Room: 304  |  Duration: 38 Sessions (Fall Semester)

This course explores fundamental computer science concepts through the lens of game development. We identify physical mechanics in board games, visualize them in Unity, and implement them in C# to solve tangible design problems.

Class Topic & Objectives Assignments / Labs
Unit 1: C# & Unity Fundamentals
01 The "Big Three": GameObject, Component, Scene
Installation verification. Understanding the component-based architecture. Navigating the Unity Editor interface. 		Lab Variables in C#. Public vs. Private. Using [SerializeField] to expose private variables to the editor (Unity Best Practice). HW Mad Libs Script: Displaying variables in the Inspector.
03 C# Basics II: Methods & Lifecycle
Writing methods. The Monobehaviour Lifecycle: Awake() vs Start() vs Update(). 		Quiz Execution Order.
04 Logic in the Loop: DeltaTime & Vectors
Using Time.deltaTime for framerate independence. Basic vector math for movement. 		Lab The Mover: Scripting smooth movement.
05 Physics & Events
Rigidbodies and Colliders. OnCollisionEnter vs OnTriggerEnter. 		Read Unity Physics Best Practices.
06 Unit 1 Capstone: Space Dodger
Combine movement, input, and collision into a simple arcade game loop. 		Due Space Dodger Build (WebGL).
Unit 2: Linear Data Structures
07 Unplugged: Searching Algorithms
Analogy: "Battleships". Linear Search vs Binary Search (O(n) vs O(log n)).
08 Arrays in C#
Fixed-size collections. Iteration with foreach. When to use Arrays (performance) vs Lists. 		Lab Weapon Spawner: Random selection from Array.
09 Lists & Dynamic Memory
The List<T> class. Adding/Removing items at runtime. Memory overhead considerations. 		HW Bullet Manager Optimization.
10 Lab: Dynamic Inventory System
Building an RPG inventory that can grow/shrink. Integrating UI elements.
11 Dictionaries & Hashing
Key/Value pairs. O(1) lookup speeds. Why Dictionaries are faster than Lists for lookups. 		Read "Spatial Hashing in Games".
12 Lab: Item Database Lookup
Creating an Item Database using ScriptableObjects and Dictionaries for instant retrieval. 		Due Database Implementation.
13 Unit 2 Review & Challenge
Refactoring "Spaghetti Code" using Arrays, Lists, and Dictionaries. 		Check Unit 2 Checkpoint.
Unit 3: Stacks & Queues (LIFO/FIFO)
14 Unplugged: The Stack
Analogy: "Tower of Hanoi" & "Pancakes". Understanding LIFO (Last-In, First-Out).
15 Stacks in Unity: State Management
Implementing UI Menu History (Back Button functionality) using Stack<GameObject>. 		Lab UI Manager Script.
16 Lab: Card Game Mechanics
Using Stacks for Draw Piles and Discard Piles. Visualizing `Pop` and `Push`.
17 Unplugged: The Queue
Analogy: "The Lunch Line". Understanding FIFO (First-In, First-Out). Enqueue vs Dequeue.
18 Queues in Unity: Turn-Based Logic
Using Coroutines and Queues to manage sequential actions (e.g., RPG battle turns). 		HW Action Queue Prototype.
19 Lab: Command Buffer System
Building an input buffer (fighting game style) using a Queue to store player key presses.
20 Priority Queues
Concept: What if some items are more important? Sorting queues. Usage in A* (preview).
21 Unit 3 Review & Challenge
Design challenge: Choose Stack vs Queue for various game scenarios. 		Check Unit 3 Checkpoint.
Unit 4: Trees, Graphs & Pathfinding
22 Unplugged: Trees & Hierarchies
Analogy: "20 Questions" and "Family Trees". Terminology: Root, Leaf, Branch, Parent, Child.
23 Trees in Code: Nodes & Recursion
Building a TreeNode class. Introduction to recursive traversal. 		HW Recursive Factorial.
24 Lab: The Skill Tree
Implementing an unlockable RPG skill tree where children nodes depend on parent nodes.
25 Unplugged: Graphs & Networks
Analogy: "Ticket to Ride" (Maps). Nodes vs Edges. Weighted vs Unweighted graphs.
26 Graph Traversal: BFS vs DFS
Breadth-First Search (Waves) vs Depth-First Search (Maze solving). Visualizing with Coroutines. 		Read Intro to Pathfinding.
27 A* Pathfinding: The Theory
Heuristics (G, H, and F costs). Why A* is the industry standard for movement.
28 Lab: A* Implementation (Part 1)
Setting up the Node Grid and Neighbor detection.
29 Lab: A* Implementation (Part 2)
Implementing the Open/Closed sets and path reconstruction loop. 		Due Pathfinding Demo.
30 Unit 4 Review & Challenge
Comparing Pathfinding algorithms. Optimizing for large maps. 		Check Unit 4 Checkpoint.
Unit 5: The Capstone Project
31 Capstone Kick-off
Theme reveal: "Synthesis". Requirements: Must use 1 Linear, 1 FIFO/LIFO, and 1 Graph/Tree structure. 		Due Team Formation & Pitch.
32 Systems Architecture
Planning the data flow. How does the Inventory (List) talk to the Skill Tree (Graph)? 		Due Architecture Diagram.
33 Development Sprint 1: Core Mechanics
Studio time. Focus on getting the primary data structures functional.
34 Development Sprint 2: Integration
Studio time. Connecting the systems together. 		Due "Greybox" Playable Build.
35 Development Sprint 3: Polish & UI
Studio time. Visualizing the data structures for the player (UI feedback).
36 Peer Review & QA
Class-wide playtesting. Bug tracking and triage. 		Due Bug Report Log.
37 Final Polish
Final build preparation. Creating the "Justification Document" explaining data structure choices.
38 Capstone Showcase
Final presentations. Playing each other's games. Course wrap-up. 		Due Final Game & Justification Doc.
